NAME
doc2ps, doc2txt, wdoc2txt, xls2txt, antiword, msexceltables, mswordstrings, olefs – read Microsoft Office documents

SYNOPSIS
doc2ps [ file.doc ]
doc2txt [ file.doc ]
wdoc2txt [ file.doc ]
xls2txt [ file.xls ]
aux/antiword [ options ] file.doc ...
aux/msexceltables [ –Dant ] [ –d delim ] [ –w worksheets ] /mnt/doc/Workbook
aux/mswordstrings /mnt/doc/WordDocument
aux/olefs [ –m mtpt ] file.doc

DESCRIPTION
The rc(1) script doc2txt uses olefs and mswordstrings to extract printable text from the body of a Microsoft Word document and write it to standard output. Wdoc2txt plumbs extracted text to a new acme(1) window. Xls2txt writes to standard output the printable text from a Microsoft Excel document.

Legacy Microsoft Office documents are stored in the Object Linking and Embedding (OLE) subset of the FAT file system format. Olefs exploits this to present the contents of an Office document as a file system at /mnt/doc (or at mtpt specified with –m). Mswordstrings or msexceltables can extract strings from the files there. Msexceltables takes the options:
–D   Print verbose debugging on standard output.
–a   Attempt conversion of non–tabular sheets (e.g., charts and graphs).
–d delim
Set the field delimiter to the string delim, by default a single space.
–
n   Do not pad fields to the column width.
–t   Truncate fields to the column width.
–w worksheets
Specify which worksheets to process. By default all tabular sheets are output. Lists of pages or page ranges may be given with individual pages separated by commas, ranges by a minus. Suppressed pages are always included in the sheet count.

Doc2ps uses antiword to write to standard output a letter–sized PostScript approximation of the Word document file.doc.

Antiword reads text, formatting, and images from the given Microsoft Word file(s) to write a representation of them to standard output. Three major options select among output modes, with sub–options unique to each mode:
–p paper
PostScript output sized to paper, one of common sheet sizes 10x14, a4, a5, b4, b5, executive, folio, legal, letter, note, quarto, statement, or tabloid. Under –p, –i level sets the handling of images to level, one of 1 (no image output), 2 (PostScript level 2, the default), 3 (PostScript level 3, experimental), or 0 (incompatible Ghostscript extensions). –L sets landscape output, horizontally oriented.
–
t   Text output (the default). Under –t, –w width breaks output lines after width number of characters.
–x dtd
XML output according to the Document Type Definition represented by dtd. Currently db, representing DocBook, is the only useful dtd code.

In all modes, –s prints `hidden' text normally suppressed by Word.

EXAMPLE
To print text from selected pages in the Excel document file.xls, delimiting unpadded output fields with @:
aux/olefs file.xls
aux/msexceltables –n –d '@' –w 1,7,9–14,3–4 /mnt/doc/Workbook
unmount /mnt/doc
The xls2txt script performs a similar procedure, modulo msexceltables options.
